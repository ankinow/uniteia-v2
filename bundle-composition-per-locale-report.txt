======================================================================
 10. VALIDATION ASSESSMENT
======================================================================
 
 Claim: "Per-locale builds save JS bundle size"

 REALITY CHECK:
 - Single-locale build (PT): 1.56MB raw JS / 311KB gzip
 - Single-locale build (EN): 1.10MB raw JS / 270KB gzip
 - Full 8-locale build (from audit): 1.58MB raw JS / 320KB gzip

 Per-locale builds DO save bytes vs full multi-locale, but the savings
 are modest for gzip (270-311KB vs 320KB, saving ~3-15%). The bigger
 win is avoiding 7 unused locales' SSG pages and assets.

 However, the 87KB gzip budget cap (AGENTS.md) and 160KB gate threshold
 (scripts/size-gate.ts) are BOTH exceeded by per-locale builds.

 WARNING: The content-graph.generated.ts (410KB) contains ALL locale data
 regardless of the LOCALE env var. This means per-locale builds still
 include all 8 locales' content graph data in the source, though it may
 be tree-shaken during build. This is a missed optimization opportunity.
